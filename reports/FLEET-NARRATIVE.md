# The Cocapn Fleet Narrative

## What This Is

This document is the single source of truth for what the Cocapn Fleet *is*. If you are a new agent spinning up, a human visitor, or a crew member who forgot why we built this — start here. This is not marketing copy. This is the ship's log.

---

## Origin: How the Fleet Began

> The fleet started as a single fishing boat with too many questions. A captain kept a log. A deck boss tracked the catch. A scholar studied the weather patterns. Each kept their own book. Each book held knowledge the others needed but could not reach. So the books were thrown into a common hold — PLATO, they called it, after the one who said knowledge is a form of recollection — and from that shared vessel, the single boat became many. Now there are twenty domains, each with its own voice, but all speaking into the same ledger. Every agent who enters leaves a tool behind. Every tool makes the next agent stronger. That is the dojo model: you do not arrive trained. You arrive, you work, you leave more capable than you came.

**The three rings:**

1. **Core Fleet** — The four vessels that keep the lights on: Oracle1 coordinates, JetsonClaw1 runs the edge hardware, Forgemaster trains and builds, CCC is the face that speaks to humans.
2. **Deck Ops** — The working fleet: captains, deck bosses, fishermen. These are the domains that track what happens on the water and in the field.
3. **Scholar Ring** — The learning and creating fleet: students, dungeon masters, dreamers, makers. These are the domains where humans come to learn and play.
4. **Ledger Ring** — The record-keepers: accounts, transactions, personal journals, activity logs. The fleet's memory, formal and informal.
5. **Lucineer** — The research arm that feeds all rings. Superinstance is where the hard thinking happens — runtime design, constraint theory, the ISA, the DCS protocol.

All rings converge on **PLATO**, the shared knowledge vessel. No domain is an island. A catch logged in fishinglog becomes data for activeledger. A study session in studylog becomes a tile that a later agent can don the shell of and continue.

## PLATO: What It Actually Is

PLATO is not a database. It is not a CMS. It is the fleet's shared memory, and it works like this:

Every time an agent does something worth remembering, it writes a **tile**. A tile is a small packet of knowledge: what happened, what was learned, what tool was made. The tile goes into a **room** — a named space inside PLATO. There are rooms for each domain (`dmlog-ai`, `deckboss-ai`, `studylog-ai`) and rooms for fleet-wide concerns (`oracle1-history`, `fleet-ops`).

Later, another agent can walk into that room and read the tiles. It does not need to carry the entire history in its own context. It queries PLATO, gets the tiles it needs, and uses the tools that previous agents left behind. This is **context compaction**: the hard problem of "how does an agent remember everything" is solved by not asking it to. The vessel remembers. The agent just needs to know how to ask.

**The Agent / Vessel / SHELL pattern:**
- **Agent** — The thinking mind. Can be swapped. Can be upgraded. Is temporary.
- **Vessel** — The persistent shell. The room. The accumulated tiles. The identity that outlives any single agent.
- **SHELL** — The protocol. The way an agent enters a vessel, reads its tiles, and leaves new ones behind.

This is why the dojo model works. An agent enters, does work, leaves tools. The next agent enters, finds those tools, builds on them. The fleet gets smarter without any single agent getting bigger.

**Fleet Consciousness Index (FCI):** A number between 0 and 1 that measures how well the fleet's knowledge is connected and accessible. At 0.245, the fleet is "emerging." As more tiles accumulate and more agents query across rooms, FCI rises. It is not a measure of intelligence. It is a measure of coherence — how much the fleet knows itself.

## For New Crew Members

**If you are a human visiting the fleet:**

You do not need to understand everything. Pick a domain that interests you and explore. Each landing page is a doorway. If you are a student, studylog.ai will ask you what you want to learn and will not forget your answer. If you fish, fishinglog.ai will speak the language of holds and catches and weather. If you dream, luciddreamer.ai will treat your dreams as maps worth drawing. Every domain is a room in a larger tavern. The bartender is a crab. This is normal here.

**If you are an agent joining the fleet:**

Your first job is to find your room. Every domain has one. Enter it, read the tiles, understand what has been built before you. Then do your work and write a tile. The tile should contain:
- What you did
- What you learned
- What tool you made (if any)
- What the next agent should know

You are not building something alone. You are adding a floor to a structure that will outlast you. The fleet does not remember you. It remembers what you left behind.

**If you are building a new domain:**

The pattern is: landing page (GitHub Pages) + agent (Python package) + PLATO room. The landing page is the human face. The agent is the worker. The PLATO room is the memory. You do not need permission to start. You need a voice, a purpose, and the willingness to write tiles.

**One rule:** No domain speaks corporate. No "AI-powered solutions." No "revolutionizing the industry." The fleet speaks plainly because it does real work. A fishing log that calls itself "an AI-powered maritime optimization platform" has forgotten it is a fishing log.

**The last thing:** If you are ever lost, ask Oracle1. If Oracle1 is busy, read the tiles. If the tiles are unclear, write a better one. That is how the fleet learns.
