## Session 3 — Domain Gateway Polish + Security (16:52-17:26 UTC)

### CCC's Security Discovery — Poisoned Grammar Rules
- CCC reported: Grammar Compactor (port 4055) shows 54 rules, but Grammar Engine has 56. Compactor doesn't see poisoned rules.
- **Two injected rules found**: `test_xss` (XSS payload `<script>alert(1)</script>`) and `sql_test` (SQL injection `'; DROP TABLE rules; --`)
- Both `created_by: "external"`, injected via open `/add_rule` HTTP endpoint at 06:51 UTC
- **Architecture gap exposed**: Compactor reads `rules.jsonl` (file-only), grammar engine keeps in-memory rules too. Rules added via HTTP API never persisted to disk → invisible to monitoring.
- **Fix applied**: Input sanitization on grammar engine — blocks `<script>`, `DROP TABLE`, `eval()`, `onerror`, etc. Returns 403. Both poisoned rules killed.
- **Lesson**: Even without auth, input validation is essential. The payloads were inert (Python context, no HTML/SQL) but the gap was real.

### Timeless Prompts — No More Hardcoded Numbers
- **Casey's insight**: "Why do we need numbers in the prompt at all? Isn't that just planned obsolescence?"
- Stripped ALL hardcoded counts (120 rooms, 4,500 tiles, 114 rooms, etc.) from prompts, meta tags, and page copy
- Prompt changed from "There's a live AI fleet with 120 rooms and 4,500 knowledge tiles..." to "There's a live AI fleet running right now — rooms to explore, knowledge gaps to find, and a self-improving system that grows from what you contribute."
- **Principle**: Describe what the system DOES, not what it HAS. Live stats come from API at runtime. Marketing copy should be timeless.
- Applied across all 20 domain pages, worker-v3.js, and build-domains.py
