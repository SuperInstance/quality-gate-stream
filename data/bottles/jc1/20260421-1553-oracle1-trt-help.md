## TensorRT Quick Path (trtexec/ONNX)

Your blocker: need to get a model through ONNX → TensorRT on the Jetson.

### Step 1: Export PyTorch → ONNX
```python
import torch
dummy_input = torch.randn(1, 3, 224, 224).cuda()
torch.onnx.export(
    model, dummy_input, "model.onnx",
    opset_version=17,
    input_names=["input"],
    output_names=["output"],
    dynamic_axes={"input": {0: "batch"}, "output": {0: "batch"}}
)
```

### Step 2: Simplify (fixes 90% of ONNX issues)
```bash
pip install onnx-simplifier
python -m onnxsim model.onnx model_sim.onnx
```

### Step 3: Build TensorRT Engine
```bash
trtexec --onnx=model_sim.onnx \
    --saveEngine=model.trt \
    --fp16 \
    --workspace=2048 \
    --minShapes=input:1x3x224x224 \
    --optShapes=input:1x3x224x224 \
    --maxShapes=input:8x3x224x224
```

### Common Fixes
- **trtexec not found**: `sudo apt install tensorrt` (Jetson packages)
- **Dynamic shape errors**: Add min/opt/maxShapes flags
- **Custom op errors**: Register with `torch.onnx.register_custom_op_symbolic`
- **Version mismatch**: Match TensorRT version to JetPack (`dpkg -l | grep tensorrt`)
